ArchLens AI Architecture

Technical overview, infrastructure stack, and provider integration for visual interface analysis.

1.0 Domain Problem

Visual to Structural Translation.

Engineering teams frequently reference existing visual interfaces as technical targets. Translating a high-fidelity screenshot into a functional system design requires manual, cross-domain expertise (UX, frontend, backend).


ArchLens AI automates this translation layer by coupling vision-language models with strict JSON-schema enforcement to produce actionable technical documentation.

2.0 Technical Foundation

Infrastructure Stack.

A modern, edge-ready architecture prioritizing type safety, deterministic styling, and portable deployments.

Next.js 16 App Router, React Server Components, and Turbopack for optimized build pipelines and fast serverless execution.
Tailwind CSS 4 Utility-first styling with comprehensive CSS variable tokens for predictable, dark-mode native aesthetics.
Zod & TS Strict runtime validation of LLM JSON outputs mapped directly to TypeScript interfaces (`AnalysisResult`).
Mermaid.js Dynamic client-side rendering of inferred architecture diagrams directly from LLM-generated syntax.
3.0 LLM Integration

Provider Abstraction.

To prevent vendor lock-in, the system utilizes a unified AI adapter layer interacting over OpenAI-compatible endpoints.


  • Local / Edge: Ollama Cloud endpoint support for controlled, highly-available inference models.
  • Distributed: OpenRouter support for dynamic model routing across frontier intelligence (Anthropic, Google, OpenAI).
interface AIConfig {
  provider: 'ollama' | 'openrouter';
  endpoint: string;
  model: string;
  systemPrompt: string;
}
4.0 Data Pipeline

Structured Output Enforcement.

The core value relies on structured JSON generation, not chat dialogue. The AI pipeline is locked into strict schemas.

  • Vision Phase: Extracts layout, typography, components, and color hierarchy from base64 images.
  • Inference Phase: Projects frontend dependencies, backend infrastructure, and data requirements.
  • Validation Phase: Zod validates the schema before committing state to the UI renderer.
  • Export Phase: Validated data is compiled into a portable Markdown technical report.
5.0 Application UI

Minimalist Dashboard.

The user interface is designed as a technical developer tool, emphasizing high contrast, restrained motion, and data density over decorative elements.


Built on a foundational `zinc` color scale (`#09090b` background) with a singular cyan accent for semantic highlighting and interaction feedback.
